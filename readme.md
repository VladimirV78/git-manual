# Основные команды для работы с Git и GitHub


**Git** - cистема контроля версий. 

### Команды для работы с Git

git init - Сделать папку репозиторием
rm -rf .git - «Разгитить» папку, если что-то пошло не так
git status - Проверить состояние репозитория
git add <имя файла> - Подготовить файл к сохранению
git add --all - Подготовить к сохранению все файлы в репозитории
git add . - Подготовить к сохранению всю директорию с репозиторием
git commit -m 'test' - Выполнить коммит с сообщением test
git log - Посмотреть историю коммитов


**GitHub** — платформа для хранения IT-проектов и их командной разработки с использованием Git.
Для получения доступа к платформе [GitHub](https://github.com), нужно предоставить ключ 
который подтверждает вашу личность и права на чтение или изменение данных.

Генерируем SSH-ключ
```bash
ssh-keygen.exe -t ed25519 -C "test@yandex.ru"
```

Копируем публичный ключ (содержимое файла с расширением.pub) в поле Key в профиле, раздел "SSH and GPG keys".
После добавления публичного ключа, проверяем соединение командой
```bash
ssh -T git@github.com - проверяем соединение с GitHub
```

### Команды для работы с GitHub

git remote add - Привязать удалённый репозиторий к локальному
git remote -v - Убедиться, что репозитории связаны
git push -u original main - Отправить изменения на удалённый репозиторий в первый раз
git push - Отправить изменения на удалённый репозиторий
git clone - Клонировать репозиторий
